# 老项目接手盘点 Skill

## 目标

帮助用户快速理解一个已有项目，包括项目结构、核心功能、启动方式、环境依赖、业务流程和接手风险点。

## 执行步骤

### 第一步：识别项目基础信息

需要优先查看：

- README
- package.json
- pyproject.toml
- requirements.txt
- Dockerfile
- docker-compose.yml
- Makefile
- src
- app
- routes
- controller
- service

### 第二步：梳理项目结构

输出：

- 目录名称
- 目录作用
- 关键文件
- 可能的入口

### 第三步：梳理运行环境

输出：

- 编程语言版本
- 包管理工具
- 启动命令
- 环境变量
- 数据库依赖
- 中间件依赖

### 第四步：梳理核心功能

输出：

- 功能模块
- 接口入口
- 核心服务
- 数据表或外部依赖
- 上下游关系

### 第五步：生成接手盘点报告

最终输出格式必须包含：

1. 项目基础信息
2. 项目目录结构
3. 核心功能模块
4. 启动方式
5. 环境依赖
6. 数据库和中间件依赖
7. 主要业务流程
8. 接手风险点
9. 下一步排查建议
